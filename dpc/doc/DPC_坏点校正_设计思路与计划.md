## 二、总体设计思路

### 2.1 核心难点分析

坏点校正的本质是：**输入像素流中，若当前像素坐标命中坏点表，则用周围邻域像素估算真实值来替换它**。

FPGA 实现有三个核心难题：

| 难点           | 问题描述                                              | 解决方向                   |
| ------------ | ------------------------------------------------- | ---------------------- |
| **坐标实时追踪**   | 我们需要知道每个像素的 (X, Y) 才能与坏点表比对                        | 像素坐标计数器，跟随帧同步信号维护       |
| **邻域像素获取**   | 校正需要周围像素（如上、下、左、右的同色像素），但视频流是单向流式的                 | 行缓冲（Line Buffer）缓存前两行数据 |
| **坏点表查找延迟**  | 1024 个条目逐条比对的延迟不可接受                                  | 排序存储 + 二分查找 / 哈希分桶     |

## 三、关键模块详细设计

### 3.1 坐标跟踪器

**功能**：为每个像素打上实时 (X, Y) 坐标标签。

**实现方式**：

```verilog
// 伪代码
always @(posedge aclk) begin
    if (!aresetn) begin
        pixel_x <= 0;
        pixel_y <= 0;
    end else if (s_axis_tvalid && s_axis_tready) begin
        if (s_axis_tuser) begin           // SOF: 新帧开始
            pixel_x <= 0;
            pixel_y <= 0;
        end else if (s_axis_tlast) begin  // EOL: 行结束
            pixel_x <= 0;
            pixel_y <= pixel_y + 1;
        end else begin
            pixel_x <= pixel_x + 1;
        end
    end
end
```

**注意事项**：
- 需要处理 BYPASS 模式下坐标仍然需要传递
- 与坏点表查询之间需要打拍对齐（pipeline register）

### 3.2 坏点表存储与查询

#### 3.2.1 存储方案选型

| 方案           | 优点             | 缺点            | 是否采用    |
| ------------ | -------------- | ------------- | ------- |
| 寄存器阵列        | 零延迟、实现简单       | 资源消耗大（LUT多）   | ❌       |
| BRAM存储       | 资源效率高          | BRAM需求 ≠ 0   | ❌（规格禁止） |
| **分布式RAM**   | 适中资源、读写快       | 容量有限          | ✅ 推荐    |
| **排序+二分查找**  | 查找快 O(logN)    | 写入慢、需预排序      | ✅ 推荐    |

**最终方案：分布式 RAM + 预排序二分查找**

- 坏点按 `{Y[11:0], X[11:0]}` 的 24-bit 复合键排序存储
- 查询时使用二分查找，查找延迟约 `log2(1024) ≈ 10` 个时钟周期
- 查找流水线：将 10 级比较拆成流水线，保证吞吐量 = 1 像素/时钟

#### 3.2.2 二分查找流水线设计

```
Cycle 1: mid = 512, 比较 cmp0 = (pixel_key >= table[512])
Cycle 2: mid = cmp0 ? 768 : 256, cmp1 = (pixel_key >= table[mid])
Cycle 3: mid = cmp1 ? ... : ... , cmp2 = ...
...
Cycle 10: 命中判断 (table[mid] == pixel_key)
```

每一级流水线处理一个 pixel，所以不影响吞吐量。总查找延迟 10 拍。

#### 3.2.3 坏点表配置接口

```verilog
// 坏点表写入 (通过寄存器接口)
// 地址 0x20: BAD_PIXEL_ADDR  (WO) - 坏点表写地址 [9:0]
// 地址 0x24: BAD_PIXEL_DATA  (WO) - 坏点坐标 {Y[11:0], X[11:0]}
// 地址 0x28: BAD_PIXEL_WEN   (WO) - 写使能 [0]
// 地址 0x2C: BAD_PIXEL_COUNT (RO) - 当前坏点数量
```

**配置流程**：
1. 复位后，坏点表为空
2. 上位机通过寄存器接口逐个写入坏点坐标
3. 写入完成后，触发排序（硬件自动排序，或要求上位机按序写入）
4. 设置 BAD_TABLE_READY 标志，开始正常处理

**简化方案**（推荐）：
- 要求上位机按坐标升序写入坏点表，FPGA 端不做排序
- 写入完成后写使能位，FPGA 直接开始二分查找
- 这样省去排序硬件，简化设计

### 3.3 行缓冲（Line Buffer）

#### 3.3.1 为什么需要行缓冲

RAW 格式下，对于 Bayer 阵列，坏点校正需要同色通道的邻域像素。以 3×3 窗口为例：

```
当前像素 (x, y) 的邻域：
  (x-2, y-2)  (x, y-2)  (x+2, y-2)    ← 需要前两行数据
  (x-2, y)    当前像素    (x+2, y)      ← 当前行
  (x-2, y+2)  (x, y+2)  (x+2, y+2)    ← 需要后两行（不可得！）
```

**关键约束**：FPGA 处理是流式的，无法预知未来的像素。因此我们只能使用**过去**的像素进行校正。

**可行邻域**（只用上行/同行像素）：

```
上两行: (x-2, y-2)  (x, y-2)  (x+2, y-2)  ← 来自 LineBuf
上一行: (x-2, y-1)  (x, y-1)  (x+2, y-1)  ← 来自 LineBuf  
当前行: (x-2, y)    [坏点]    (x+2,  y)   ← 当前行（x+2还未知）
```

**实际上**：对于 Bayer RAW，同色像素间距为 2。所以真正可用的同色邻域：

```
上方同色:  (x, y-2)               ← 需要 2 行缓冲
左方同色:  (x-2, y)               ← 当前行寄存器即可
```

#### 3.3.2 行缓冲实现

```verilog
// 2行缓冲（BRAM规格说0块，这里用分布式RAM，但实际可能需要BRAM）
// 权衡：若严格遵循0 BRAM，则分辨率上限需要降低，或简化为仅用同行邻域

// 方案A：简化校正 - 仅使用同行左/右邻域 + 上行同色邻域 (1行缓冲)
//   LineBuf × 1, 深度=MAX_WIDTH, 位宽=DATA_WIDTH

// 方案B：完整3×3邻域 (2行缓冲) 
//   LineBuf × 2, 深度=MAX_WIDTH, 位宽=DATA_WIDTH
//   BRAM估算: 2 × 3840 × 12 ≈ 92 Kbits ≈ 10 块BRAM
//   与"0块BRAM"要求冲突！

// 推荐方案A：单行缓冲 + 同行邻域插值
```

### 3.4 校正算法

#### 3.4.1 算法选择

针对 RAW 格式 Bayer 阵列的坏点校正：

```
┌──────────────────────────────────────┐
│   Bayer RGGB 阵列示意                 │
│                                      │
│   R  G  R  G  R  G                  │
│   G  B  G  B  G  B                  │
│   R  G [R] G  R  G   ← [R] 是坏点    │
│   G  B  G  B  G  B                  │
│   R  G  R  G  R  G                  │
│                                      │
│   同色邻域 (R通道):                    │
│   - 上方: (x, y-2)                   │
│   - 下方: (x, y+2) — 不可用           │
│   - 左方: (x-2, y)                   │
│   - 右方: (x+2, y)                   │
└──────────────────────────────────────┘
```

**校正公式**：

**方法一：邻域平均**
```
若仅左/上可用:
  corrected = (pixel_left + pixel_up) / 2

若仅左可用:
  corrected = pixel_left

若左/右/上可用:
  corrected = (pixel_left + pixel_right + pixel_up) / 3
```

**方法二：线性插值**
```
corrected = pixel_left + (pixel_right - pixel_left) / 2
         = (pixel_left + pixel_right) / 2
# 与邻域平均本质上等价
```

#### 3.4.2 算法状态机

```verilog
// 坏点校正FSM
localparam CORR_NONE   = 2'd0;  // 不是坏点，直通
localparam CORR_AVG_LU = 2'd1;  // 左+上平均
localparam CORR_AVG_LR = 2'd2;  // 左+右平均
localparam CORR_AVG_L  = 2'd3;  // 仅左值

always @(*) begin
    case (corr_mode)
        CORR_NONE:   m_pixel = s_pixel;
        CORR_AVG_LU: m_pixel = (left_pixel + up_pixel) >> 1;
        CORR_AVG_LR: m_pixel = (left_pixel + right_pixel) >> 1;
        CORR_AVG_L:  m_pixel = left_pixel;
        default:     m_pixel = s_pixel;
    endcase
end
```

**边界处理**：
- 图像左边界 (x = 0)：无左邻域 → 使用上行邻域或保持原值
- 图像右边界 (x = W-1)：无右邻域 → 仅用左/上
- 图像上边界 (y = 0)：无上行邻域 → 仅用同行邻域
- 首行首位：无任何邻域 → 保持原值（无法校正）

### 3.5 BYPASS 模式

当 BYPASS 模式使能时：
- 输入像素直接透传至输出
- 坐标跟踪和坏点表查询仍然运行（便于调试）
- 状态寄存器可监控命中次数，但不执行替换

## 四、寄存器定义

| 偏移    | 寄存器名              | 类型  | 说明                                |
| ----- | ----------------- | --- | --------------------------------- |
| 0x00  | IP_VERSION        | RO  | IP版本号                             |
| 0x04  | IP_CTRL           | RW  | [0]使能 [1]旁路 [2]调试 [3]坏点表就绪       |
| 0x08  | IP_STATUS         | RO  | [15:0]坏点命中计数                       |
| 0x0C  | IP_RESET          | WO  | 写1复位统计                           |
| 0x10  | IMG_WIDTH         | RW  | 图像宽度                              |
| 0x14  | IMG_HEIGHT        | RW  | 图像高度                              |
| 0x18  | IMG_FORMAT        | RW  | [1:0] 00=RAW8 01=RAW10 10=RAW12 |
| 0x1C  | CORR_THRESHOLD    | RW  | 校正阈值（邻域差异超过此值才校正）                |
| 0x20  | BAD_TABLE_ADDR    | WO  | 坏点表写入地址 [9:0]                      |
| 0x24  | BAD_TABLE_DATA    | WO  | 坏点表数据 {Y[11:0], X[11:0]}          |
| 0x28  | BAD_TABLE_WR      | WO  | [0]写使能 [1]表锁定                     |
| 0x2C  | BAD_TABLE_COUNT   | RO  | 当前坏点表条目数                          |
| 0x30  | BAD_HIT_COUNT     | RO  | 累计坏点命中次数                          |
| 0x34  | FRAME_COUNT       | RO  | 处理帧数计数                            |

## 七、技术风险与应对

| 风险                          | 影响  | 概率  | 应对措施                                    |
| --------------------------- | --- | --- | --------------------------------------- |
| 二分查找流水线深度过大（10级），导致总延迟超标        | 中   | 中   | 采用哈希分桶：按高N位(Y)分成2^N个桶，桶内线性查找，降为2~3级流水线 |
| 1行缓冲资源超限（分布式RAM不够放3840×12bit） | 高   | 中   | 降级为仅使用同行邻域校正（无行缓冲），或使用少量BRAM（需申请例外）    |
| RAW格式Bayer同色邻域间距为2，校正效果不佳      | 中   | 低   | 使用距离加权插值代替简单平均；引入非线性校正阈值判断              |
| 坏点表1024条，二分查找流水线资源占用过大          | 低   | 中   | 减小流水线深度，采用分段流水线（多周期吞吐但保持阶段流水）            |

## 八、简化实现路线（最小可行方案 MVP）

若时间紧张，可按以下最小方案先跑通全流程，再迭代优化：

| 组件        | MVP 方案                     | 后续优化                |
| --------- | -------------------------- | ------------------- |
| 坏点表存储     | 寄存器阵列（≤64个坏点）              | 分布式RAM + 二分查找（1024个） |
| 坏点表查找     | 寄存器逐条比较（组合逻辑，1拍完成）         | 流水线二分查找              |
| 行缓冲       | 无行缓冲，仅用同行左/右邻域             | 1行或2行缓冲              |
| 校正算法      | 仅邻域平均                      | 增加线性插值 + 阈值判断        |
| 边界处理      | 边界坏点保持原值                   | 边界不对称校正              |

## 九、关键技术决策记录

| 决策点     | 选项A            | 选项B          | 选择      | 理由              |
| ------- | -------------- | ------------ | ------- | --------------- |
| 坏点表存储   | 分布式RAM + 二分查找  | 寄存器阵列 + 线性查找 | **A**   | 支持1024条目，查找延迟可控  |
| 行缓冲数量   | 0行（仅同行邻域）      | 1行缓冲         | **B**（灵活） | MVP用A，优化用B       |
| 校正算法    | 邻域平均           | 线性插值         | **A**   | 实现简单，效果等价        |
| 坏点表写入   | 要求上位机预排序写入     | FPGA端自动排序     | **A**   | 简化硬件，配置由软件负责    |
| 坏点命中统计  | 包含              | 不包含          | **A**   | 便于调试和效果评估        |

## 十、总结

缺陷像素校正 IP 的核心设计思路可以概括为 **"追踪→查表→校正"** 三步走：

1. **追踪**：用坐标计数器实时追踪每个像素的 (X, Y) 位置
2. **查表**：用流水线二分查找将像素坐标与预存坏点表比对
3. **校正**：命中坏点后，用邻域同色像素的平均值替换原值

整体流水线延迟控制在 **10~12 拍以内**，满足 1 像素/时钟的吞吐要求。建议**先用 MVP 方案跑通全流程**，再逐步优化到完整规格。
